# Ohaka Navi Sitewide UI Directions

**Date:** 2026-04-18  
**Scope:** `ohakanavi.jp` 全体のサイトデザイン改善方針  
**Objective:** 速度改善と両立しながら、世界トップクラスの UI 品質に引き上げるための全ページ共通アートディレクションを定義する

---

## 現状評価

現状のサイトは、情報量と導線は一定以上ある一方で、ページ群ごとのデザイン思想が揃っていない。

主な課題は次のとおり。

- ホーム、検索、診断、ガイド、詳細、問い合わせでトーンが分散している
- カード依存が強く、プレミアム感より「整理された比較サイト」に見えやすい
- CTA の強弱が弱く、重要な行動が視覚的に埋もれやすい
- 写真、余白、タイポグラフィの優先順位がページごとに不統一
- 「中立に相談できる安心感」と「上質なブランド体験」が同時に成立していない

世界トップクラスを目指すなら、まず必要なのは部分改善ではなく、全ページを束ねる明確なデザイン思想である。

---

## 推奨方針

最も推奨するのは、**Direction A を基軸**にして、**Direction B の相談体験設計を要所に混ぜる**方法。

理由は次のとおり。

- `ohakanavi` は価格訴求よりも信頼・品格・安心が重要
- 供養というテーマ上、派手さより静かな強さのほうが長期的にブランド資産になる
- ただし静かすぎると CV が弱くなるため、相談導線には人の温度が必要

---

## Direction A: Quiet Luxury Editorial

### コンセプト

高級宿泊ブランド、建築的なギャラリーサイト、上質なエディトリアルの中間。
「静かな高級感」「余白」「写真の品格」「少ない言葉で伝える強さ」を軸にする。

### ビジュアル方針

- ヒーローは full-bleed 写真または静かな映像に近い構成
- タイポグラフィは明朝を主役、サンセリフを補助にする
- 配色は現行の和色系をベースに、彩度を抑えてコントラストで見せる
- カードは最小限にし、区切りは余白・罫線・背景面の切り替えで作る
- ボタンは数を減らし、主 CTA のみ存在感を強くする

### ページ別適用

- `/`
  ヒーローをポスター化。検索 UI は 1 枚の美しい面として統合する
- `/search`
  不動産一覧ではなく、上質なセレクション体験に寄せる
- `/detail/[id]`
  情報の羅列ではなく、写真・立地・供養条件を静かに見せる
- `/guide/*`
  雑誌記事のような読み味を持たせる
- `/contact`, `/consult/*`
  入力フォームの圧を減らし、安心感を優先する

### 向いているブランド印象

- 信頼
- 中立
- 上質
- 誠実

### リスク

- 実装が甘いと「地味」になる
- 画像選定の質が UI 品質を大きく左右する

---

## Direction B: Warm Human Concierge

### コンセプト

「検索サイト」ではなく「供養の相談ブランド」として見せる。
人が寄り添う温度、相談の安心感、やわらかな導線設計を主軸にする。

### ビジュアル方針

- 写真は人物、手元、相談風景、寺院空間など温度のあるものを使う
- 色はあたたかい生成り、深い青緑、落ち着いた朱を主にする
- フォームや CTA は丸みを持たせるが、安っぽくしない
- 診断や問い合わせは「作業」ではなく「対話」に見せる
- 小さな安心要素を増やす
  - 回答目安時間
  - 相談無料
  - しつこい営業なし
  - 専門スタッフ対応

### ページ別適用

- `/consult/*`
  最も相性が良い。温度感のある LP にできる
- `/contact`
  単純なフォームから相談窓口へ格上げできる
- `/choices/diagnosis`
  ツール感を減らして会話体験にできる
- `/partner/contact`
  法人向けでも丁寧なブランド印象が作れる

### 向いているブランド印象

- 親切
- 安心
- 人間味
- 相談しやすさ

### リスク

- 相談導線に寄りすぎると検索・比較サイトとしての強みが弱く見える
- ビジュアルの温度感が過剰だと保険営業風になる

---

## Direction C: Modern Cultural Platform

### コンセプト

供養・お墓・墓じまいを扱う「知的なプラットフォーム」として再構成する。
検索、ガイド、診断、比較、相談を 1 つの洗練された知識サービスとして見せる。

### ビジュアル方針

- グリッドとタイポを強く使う
- 情報設計の見やすさを美しさに変える
- カルチャーメディアやハイエンドなナレッジプロダクトのように見せる
- セクションごとに明確な役割を持たせる
- 余白と見出し階層で読む体験をコントロールする

### ページ別適用

- `/guide`
  最も相性が良い。記事群の価値が上がる
- `/choices/*`
  比較コンテンツを知的に見せやすい
- `/search`
  フィルタ UI を洗練されたプロダクト体験にできる
- `/detail/[id]`
  データと情緒を両立しやすい

### 向いているブランド印象

- 専門性
- 現代性
- 体系性
- メディア資産

### リスク

- 冷たく見えると、供養テーマに必要な優しさが不足する
- 相談への導線を補強しないと CV が落ちる可能性がある

## ページ群ごとの推奨適用

### 1. ホーム

推奨: Direction A ベース、CTA 導線のみ B を混ぜる

- 第一印象は上質で静か
- 主要アクションは「探す」と「相談する」の 2 軸に限定
- 特集カードの量を減らし、ヒーローと信頼訴求に重心を置く

### 2. 検索一覧

推奨: A と C のハイブリッド

- 情報密度は高くてよいが、賑やかに見せない
- フィルタ、並び替え、比較導線は整然と
- 写真と条件タグに主従をつける

### 3. 霊園・寺院詳細

推奨: A

- 写真の見せ方を最重要視
- 価格、立地、供養方式、安心材料を視線順に並べる
- CTA は 2 個までに絞る

### 4. 診断ページ

推奨: B と C のハイブリッド

- ウィザード UI を会話体験へ寄せる
- 質問ごとの心理的負荷を下げる
- 結果表示は「ランキング」ではなく「納得できる提案」にする

### 5. ガイド・コラム群

推奨: C ベース、見出しや写真は A

- 記事一覧はメディアとして整理
- 記事詳細は読みやすさ最優先
- CTA は文脈に合わせて控えめに配置

### 6. 相談・問い合わせ・資料請求

推奨: B

- 送信フォーム感を減らす
- 相談窓口としての安心材料を先に見せる
- 入力項目は説明とセットで迷わせない

## 共通デザインルール

### 1. Typography

- 主役書体: 品のある和文明朝
- 補助書体: 可読性の高いサンセリフ
- 見出しサイズ差を大きくし、本文は小さくしすぎない
- 強調は太字ではなく余白と階層で行う

### 2. Color

- ベースは生成り、墨、青灰、鈍い緑
- アクセントは 1 色を主軸にする
- 色数で賑やかにしない
- 成約導線色と情報強調色を分ける

### 3. Layout

- カード乱用を止める
- 余白で高級感を作る
- セクションごとに 1 つの主役だけを置く
- ヒーローは「説明」ではなく「印象」を担う

### 4. Motion

- ヒーローに 1 つ
- スクロール連動に 1 つ
- CTA または hover に 1 つ
- 全体で 2〜3 種に抑える

### 5. Imagery

- 実在感のある写真を優先
- テキストを載せる余白がある写真を選ぶ
- 看板や文字が多い写真は避ける
- 世界観の統一が最優先

## 最終提案

`ohakanavi` の最適解は以下。

- ベース世界観: **Direction A**
- 相談導線の設計: **Direction B**
- ガイド / 比較 / 検索の情報設計: **Direction C**

つまり、

**「静かで上質なブランド体験」**
に、
**「相談しやすい温度」**
と
**「知的で整理された情報設計」**
を重ねるのが最適。

---

## 次アクション

実装に進む場合は次の順番を推奨する。

1. ホームのアートディレクションを決める
2. サイト共通の typography / spacing / CTA ルールを再定義する
3. `Navbar`, `Footer`, `Hero`, `Section heading`, `CTA block` を共通化する
4. `search`, `detail`, `guide`, `consult` の順で横展開する
