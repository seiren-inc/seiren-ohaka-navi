# 要件定義書 — seiren-ohaka-navi

> Project: お墓探しナビ
> Domain: ohakanavi.jp
> Version: v1.0
> 最終更新: 2026-02-24

---

## 1. 目的

全国の墓地・霊園・寺院を検索できるポータルを構築し、
ユーザーが最適なお墓を比較・検討・問い合わせできる環境を提供する。

単なる情報掲載サイトではなく、
資料請求・相談・改葬サポートまでを一貫導線で設計する。

---

## 2. 想定ユーザー

### 2.1 エンドユーザー
- 40〜80代
- 親の墓を探している
- 生前墓を検討している
- 墓じまいを検討している
- 改葬許可申請を必要としている

### 2.2 掲載事業者
- 寺院
- 霊園管理者
- 石材店
- 納骨堂運営者

### 2.3 管理者
- 清蓮運営チーム
- 将来的には営業担当も利用

---

## 3. 必須機能要件

### 3.1 検索機能
- 都道府県検索
- 市区町村検索
- エリア別ルーティング
- 条件絞り込み（将来拡張）

### 3.2 施設詳細ページ

表示項目：
- 施設名
- 所在地
- アクセス
- 価格帯
- 写真
- プラン一覧
- 問い合わせボタン
- 資料請求ボタン

### 3.3 資料請求フォーム

2ステップ構成

Step1：
- 名前
- 電話番号
- メール

Step2：
- 希望エリア
- 相談内容
- 備考

要件：
- 二重送信防止
- 保存保証
- 完了画面表示
- 管理画面反映

### 3.4 墓じまい相談フォーム

- 改葬予定地
- 現在の墓所在地
- 期限
- 相談内容

保存保証必須。

### 3.5 管理画面
- 施設CRUD
- プランCRUD
- 問い合わせ一覧
- ステータス管理
- 画像アップロード
- Storage連携

### 3.6 改葬許可データ取得構想
- 自治体リンク管理
- PDF管理
- linkStatus制御
- PDF_ONLY管理ルール厳守
