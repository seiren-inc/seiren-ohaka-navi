## 0. 目的

本設計書は、お墓探しナビにおける
データ分析基盤、ユーザー行動解析、CVR改善、ABテスト運用、
およびグロース戦略の設計を定義する。

目標：

1. 数値に基づく改善
2. CVR最大化
3. SEO改善の定量評価
4. 収益最大化の再現性確保
5. 勘や感覚に依存しない運営

---

## 1. 分析の全体構造

### 1.1 データレイヤー

| Layer | 対象 |
|-------|------|
| Layer 1: 行動ログ | ページ閲覧、検索クエリ、フィルタ利用、CTAクリック、フォーム遷移 |
| Layer 2: コンバージョン | 問い合わせ送信、資料請求送信、有効リード判定 |
| Layer 3: 収益 | 有料掲載、リード課金、継続率 |

---

## 2. 計測対象イベント設計

### 2.1 ページ系

| イベント | パラメータ |
|---------|-----------|
| page_view | page_type, slug, referrer |

### 2.2 検索系

| イベント | パラメータ |
|---------|-----------|
| search_submit | query, normalized_query, result_count |
| filter_apply | filter_type, filter_value |

### 2.3 施設詳細

| イベント | パラメータ |
|---------|-----------|
| facility_view | facility_id, prefecture, city |
| facility_cta_click | facility_id, cta_type (inquiry/request/phone) |

### 2.4 フォーム

| イベント | 説明 |
|---------|------|
| form_start | フォーム入力開始 |
| form_complete | 送信完了 |
| form_error | エラー発生 |
| duplicate_submit_blocked | 二重送信ブロック |

## 3. KPIトラッキング設計

### 3.1 SEO指標

- オーガニック流入
- インデックス数
- 上位キーワード数
- 平均掲載順位
- CTR（検索結果→流入）

### 3.2 UX指標

- 施設詳細閲覧率
- 詳細→CTA率
- フォーム完了率
- 離脱率
- 滞在時間

### 3.3 収益指標

- リード数
- 有効リード率
- 施設あたりCVR
- 有料掲載率
- LTV（将来）

## 4. データ保存設計

### 4.1 ログ保存方針

軽量化のため：
- 行動ログは将来BigQuery等へ連携可能設計
- DB肥大化防止

| フェーズ | 方式 |
|---------|------|
| 初期 | Google Analytics + Search Console |
| 将来 | 独自ログテーブル（SearchLog, EventLog）|

## 5. ダッシュボード設計

- 地域別CVR

- 離脱ページランキング

## 6. ABテスト設計

### 6.1 対象例

- CTAボタン文言
- CTA配置位置
- 施設詳細レイアウト
- 記事からの導線

### 6.2 実施ルール

- 期間最低2週間
- サンプル数基準を設定
- 1回に1要素のみ変更
- 勝者のみ採用

禁止：
- 同時複数変更
- 数値確認前の判断
